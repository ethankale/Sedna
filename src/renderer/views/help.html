GData 2

The Basics

GData 2 is a database for managing environmental data, with an emphasis on hydrologic and metorologic data. It can handle continuous data, like 15 minute stream or rainfall records, and discrete data, like water quality samples.

GData 2 is public domain. You are free to use or modify it as you see fit.

The original GData was created by Brendan Grant and others at King County, Washington. GData 2 was created by Nat Kale at Thurston County, Washington. The design of GData 2 is influenced by the paper Towards a water quality database for raw and validated data with emphasis on structured metadata.

Getting Started

If you're this far you've already installed the application from the GData 2 GitHub. The next thing to do is to set up an SQL Server instance with an authorized user, put the user settings into GData 2, then load data (either from an instance of GData or directly through the interface).

SQL Server Installation and Setup

To install the application:

  1. Get access to an MSSQL instance (2012 or above). You can install the Express version on your desktop from Microsoft.
  2. Install SQL Server Management Studio (SSMS). Download from the Microsoft website.
  3. Open a new Query window in SSMS and run the database install/setup scripts in the following order:
    1. ./src/sql/schema.sql
    2. ./src/sql/load_from_gdata.sql (optional, if you're migrating from GData. Only works if the GData database is on the same server as the Sedna database)
    3. ./src/sql/indexes.sql
  4. Use SSMS to set up a user with read/write access on the newly created database.
    • This user can't use the windows login; it has to use a password login.
    • Enable TCP connections for the server in the SQL Server Configuration Manager. See this Stack Overflow post..
    • Enable SQL Server logins; by default it only allows Windows connections. See this stack overflow post..
  5. Download the program (currently, grab it from Github. No formal distributions yet).
  6. Open the program, immediate go to settings, and put in the SQL Server info for the DB and the user you created.
That should do it. The most likely issue you'll run into is with the user; make sure the user you create and use has permission to read, write, and delete for every table in the database.

Adding Metadata

Before you can add date, you need at least one of each of the following:

  • Site
  • Sample Point (belongs to a Site, so create a Site first)
  • Parameter
  • Method
  • Unit
You can add all of these through the Metadata page.
